Understanding ES6

Introduction of ES6

What is ES6

  • ECMAScript 2015, also well known as ES6 is a standard specification published in 2015.
    *see the link below to check availablity for each browser.
  • There are some of way to use function if your browser does not support. these are the example.
    1. Use transpile - convert from the formula written by ES6 to the other formula which can be active on the browser.
      Babel is most popular example.
    2. Use Polyfill library - Polyfill library is the library which supplements missing functions on your browser.
      By loading this in advance, polyfill library will be replaced instead of unimplemented functions.
      this is the most popular example: core-js

let

let is a reassign-able variable.

Example

Input
      
        let message = "Hello ES6";
        console.log(message);
        message = "Goodbye ES5";//overwriting "Hello ES6"
        console.log(message);
Output(on console)
      
        Hello ES6
        Goodbye ES5

const

const is a not reassign-able variable.
*const is not a property but read-only reference. Therefore, object can be changed.

Example

Input
      
        const message2 = "Hello ES6";
        console.log(message2);
        message2 = "Goodbye ES5";
        console.log(message2);//uncaught typeerror will shown
Output(on console)
      
        Hello ES6
        Uncaught ReferenceError: message is not defined

The scope of const

Scope of Let and Const is calling as "Blocked scope" generally. Blocked scope is surrounded by { }.

On the other hand, the block of var is called "function scope".
let and const is much more secure than var. If the declaration of let or const has already duplicated, this kind of error will appear below.

Duplicate declaration "name"

Input
      
        const name = "SZDP Design";
        if (true){
        //ここはブロック内なのでnameは別物
          const name = "Old SZDP Design";
          console.log("-----if block-----");
          console.log(name);//Old SZDP Design
          console.log("------------------")
        }
        console.log(name);//SZDP Design
Output(on console)
      
        -----if block-----
        Old SZDP Design
        ------------------
        SZDP Design

Caution of using const

const is not a property but read-only reference. Therefore, object can be changed.
if you want to freeze specified object, useobject.freeze

Input
      
        const project = {
          name: "SZDP v2018.09"
        }
        console.log(project.name);//SZDP v2018.09
        //you can change name of object property
        project.name = "SZDP v2018.10";
        console.log(project.name);//SZDP v2018.10
        //but object's own reference can't change
        project = {
          name: "SZDP v2019"
        }
        console.log(project.name);//Uncaught TypeError: Assignment to constant variable.
Output(on console)
      
        SZDP v2018.09
        SZDP v2018.10
        Uncaught TypeError: Assignment to constant variable.
